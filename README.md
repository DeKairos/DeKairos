### ◈ &nbsp; The Mind

I'm Shiv — a CS undergrad two years into understanding why the field is far stranger and more beautiful than any intro course suggests.

I gravitate toward problems at the intersection of disciplines: audio signal processing that turns mathematics into animation, machine learning that turns data into taste, 3D visualizations that turn abstractions into something you can spin with your mouse.

My instinct when encountering something I don't understand is to build a small version of it until I do. Two years of doing that has only made the list of things I want to build longer.

### ◈ &nbsp; Investigations

<img src="https://readme-typing-svg.demolab.com?font=Share+Tech+Mono&size=13&duration=4000&pause=2000&color=00C9B1&center=false&vCenter=true&width=500&lines=%E2%86%92+Five+problems.+Five+different+ways+of+asking+%22what+if%3F%22" alt="subheading" />

<br/>

**`001` &nbsp; The Sound That Lives in Your Taskbar**

*Taskbar Audio Visualizer — Python, PyQt6, FFT, Win32 APIs*

The question: what if your taskbar responded to music the way your body does — not with data, but with feeling? The answer involved FFT pipelines, beat detection, transparent click-through overlays, and a Win32 API rabbit hole that took me further into the OS than I expected. Renders at 33 FPS. Themes itself from your album art.

`PyQt6` &nbsp; `NumPy` &nbsp; `soundcard` &nbsp; `Win32 API` &nbsp; `FFT` &nbsp; `Inno Setup`

<br/>

**`002` &nbsp; The Globe That Knows Things**

*Globe Connect — React.js, Three.js, WebGL*

What happens when you project live data onto a spinning 3D sphere and let people explore it? Turns out: you learn a lot about the gap between information and comprehension. This was really a question about interface design — how do you make a dataset feel like a *place*?

`React.js` &nbsp; `Three.js` &nbsp; `WebGL` &nbsp; `REST APIs`

<br/>

**`003` &nbsp; Teaching a Machine to Cook**

*Recipe Recommendation System — Python, Scikit-learn, NLP, Pandas*

Given a handful of ingredients, can a model understand what you're trying to make? This became an NLP problem more than an ML one — recipes are written for humans, not machines. Similarity-based matching got it to a place where the recommendations felt like suggestions, not results.

`scikit-learn` &nbsp; `Pandas` &nbsp; `NLP` &nbsp; `Cosine Similarity`

<br/>

**`004` &nbsp; How Much Is This Worth?**

*Price Prediction Model — TensorFlow, Keras*

A structured benchmark: put Linear Regression, Decision Trees, and a Neural Network in the same ring and see what wins. The real insight wasn't the winning model — it was understanding *why* each one fails differently. Feature scaling and hyperparameter tuning sharpened the final answer.

`TensorFlow` &nbsp; `Keras` &nbsp; `scikit-learn` &nbsp; `Feature Engineering`

<br/>

**`005` &nbsp; How Complicated Is This, Really?**

*Code Complexity Analyzer — Python, AST*

Developers have intuitions about code quality. This tool makes those intuitions measurable — computing cyclomatic complexity and maintainability indices so that "this function is too complex" becomes a number, not just a feeling.

`Python` &nbsp; `AST` &nbsp; `Cyclomatic Complexity` &nbsp; `Static Analysis`
